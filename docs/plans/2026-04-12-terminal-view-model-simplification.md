# Terminal View Model Simplification

目标是把终端交互收成两个稳定概念：谁在控制、这台设备如何观看。真实终端尺寸仍然只有一套，但不同设备的本地显示策略自动处理，不再让用户自己理解 `watch`、`fit`、`full screen` 之间的内部关系。

这轮改动分三块：

1. 统一控制文案和入口
   - 把分散的 `Watch/Control/Take Control/Release/Fit Here` 收成更清楚的一组文案
   - 减少重复状态提示，去掉会制造认知负担的冗余 badge

2. 给最大化终端加本地自适应显示
   - 手机上看宽终端时自动按宽度缩放到 100%
   - 桌面上看窄终端时保持真实宽度并居中，不把终端强行铺满
   - 这些只改本地显示，不改真实 PTY 尺寸

3. 补回归测试和 e2e
   - 纯逻辑测试覆盖文案和本地展示策略
   - e2e 覆盖跨设备 handoff 后的本地渲染行为和新文案
